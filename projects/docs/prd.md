# 哄哄模拟器 — 产品需求文档 (PRD)

## 一、问题陈述

情侣吵架后，面对正在生气的另一半，脑子里一片空白——不知道该说什么话、哪些话有用、哪些话会踩雷。「哄哄模拟器」通过选择题模拟吵架场景，让用户在安全环境中练习"哄人话术"，同时享受奇葩选项带来的搞笑体验。

**核心卡点：** 面对正在生气的人，不知道该说什么话、哪些话会踩雷。

**目标用户：** 两类各半——认真学哄人的人 & 找乐子玩梗的人，产品设计需同时服务两者。

---

## 二、游戏流程

1. 用户选择对方性别（女朋友 / 男朋友）
2. 用户从预设场景中选一个，进入游戏
3. AI 生成对方说的第一句话 + 6 个回复选项
4. 用户选一个选项，AI 根据选择动态生成下一轮对话 + 新的 6 个选项
5. 如此循环，直到游戏结束

### 提前结束规则

- 好感度达到 80 分时**当轮立刻胜利结束**，不需要等 10 轮打满
- 好感度跌到 -50 以下时**当轮立刻失败结束**
- 第 10 轮结束时好感度仍未达 80 也算失败

---

## 三、预设场景

1. **忘记纪念日** —— 今天是你们在一起三周年，你完全忘了
2. **深夜不回消息** —— 你昨晚打游戏到凌晨三点，对方发了十几条消息你都没回
3. **被发现和异性聊天** —— 对方看到你和异性朋友的暧昧聊天记录
4. **把对方的猫弄丢了** —— 你帮对方照顾猫的时候，猫跑丢了
5. **当众让对方没面子** —— 你在朋友聚会上开了一个过分的玩笑

---

## 四、好感度系统

- 初始值：20 分（满分 100，最低 -50）
- 胜利条件：10 轮内好感度 >= 80（达到即立刻胜利）
- 失败条件：好感度降到 -50 以下（立刻失败），或 10 轮用完好感度仍 < 80
- 分值对用户隐藏具体加减数字，只展示当前总分和进度条
- 不要通过任何方式提示用户不同选项的潜在影响

### 情绪区间映射

| 好感度 | 对方情绪表现 |
|--------|-------------|
| -50 ~ 0 | 非常生气，冷暴力或激烈质问 |
| 0 ~ 30 | 还在生气，但愿意听你说 |
| 30 ~ 60 | 开始软化，嘴上生气但语气缓和 |
| 60 ~ 80 | 快被哄好了，可能撒娇或小声说"哼" |
| 80+ | 原谅了，但还要你保证不再犯 |

---

## 五、选项设计（每轮 6 个）

- 2 个**加分选项**（+5 到 +20）：真诚道歉 / 具体弥补方案 / 提起共同回忆等
- 4 个**减分选项**（-5 到 -30），其中：
  - 1~2 个**普通减分**：敷衍、转移话题、找借口
  - 2~3 个**奇葩搞笑选项**：离谱到好笑，例如"要不我给你跳一段？"、"我错了，但你也有错啊你不应该生气"、"我请你吃肯德基疯狂星期四行不行"
- 选项顺序随机打乱，不要把加分项固定在某个位置
- 选项如果超过一行，需要换行显示
- 单个选项分值上限 ±30，超出需 clamp 到边界

---

## 六、动态生成规则

- 不预设题库，每一轮都通过 LLM 动态生成
- 对方说的话必须和前面的对话连贯，模拟同一段连续对话
- 不能出现重复的题目和选项
- 对方的情绪要根据好感度自然变化（参见情绪区间映射表）

### LLM 调用上下文策略

每轮调用 LLM 时传入三部分：

1. **系统指令**：游戏规则 + 角色设定 + 选项生成规则 + 当前好感度 / 轮次 / 情绪区间
2. **完整历史对话**：对方说了什么 + 用户选了什么（最多 10 轮，全量传入，不截断）
3. **本轮用户选择**：用户刚选的选项文字

### LLM 输出格式与异常处理

**输出格式（JSON）：**

```json
{
  "reply": "对方说的话",
  "emotion_score_change": 10,
  "options": [
    { "text": "选项文字", "score_change": 15, "is_correct": true },
    { "text": "选项文字", "score_change": -10, "is_correct": false },
    { "text": "选项文字", "score_change": -25, "is_correct": false },
    { "text": "选项文字", "score_change": 5, "is_correct": true },
    { "text": "选项文字", "score_change": -5, "is_correct": false },
    { "text": "选项文字", "score_change": -20, "is_correct": false }
  ]
}
```

**兜底策略（三层）：**

1. **格式校验**：JSON 解析失败 → 同一 prompt 重试一次
2. **数量校验**：选项不足 6 个 → 重试；超过 6 个 → 截取前 6 个
3. **分值校验**：单选项分值超 ±30 → clamp 到边界值

重试一次仍失败 → 返回预设通用默认选项，保证游戏不卡住。

## 七、界面要求

- 像微信私聊界面：对方的话在左侧，用户选择的话在右侧
- 对话双方需要有头像（用 SVG 模拟，对方用可爱卡通头像，用户用简单头像）
- 顶部显示：好感度进度条 + 当前轮次（如"第 3/10 轮"）
- 好感度变化时有动画效果（加分绿色上升，减分红色下降）
- 适配电脑和手机浏览器，响应式布局
- 尽可能增加动效（选项飞入、头像弹跳、气泡动画等）

---

## 八、语音功能（TTS）

### 触发范围

- 对方的每一轮回复均生成 TTS 语音
- 游戏结束时对方的最后一句话也生成 TTS 语音

### 声音选择

游戏开始前让用户选择对方的声音类型：

- 女友模式：温柔女声 / 霸道御姐 / 可爱软妹
- 男友模式：低沉男声 / 温柔男声

### 播放策略

- 语音采用**异步加载**：文字先显示，语音生成完毕后播放按钮变为可点击状态
- 用户进入下一轮时**自动打断前一条语音**，立即播放新一轮语音
- 前端维护单一 audio 实例，新语音触发时先 pause 旧的再 play 新的
- 语音要体现情绪变化（根据好感度区间调整语调）

### 技术实现

- 使用扣子编程沙箱环境内置的 TTS 能力

---

## 九、游戏结束

### 成功

- 撒花动画
- 对方说一句甜蜜的话 + 语音播放
- **生成结果卡片**按钮：一键合成包含结局文字 + 好感度 + 场景名 + 金句的分享图片，用户可长按 / 右键保存
- **分享给朋友**按钮：生成带场景参数的链接（如 `?scene=anniversary`），朋友打开后直接进入该场景选择页
- 文案："通关！分享给朋友试试？"

### 失败

- 心碎动画
- 对方说一句绝情的话 + 语音播放
- "再试一次？"按钮（可重试同一场景，LLM 重新生成）

---

## 十、文风

俏皮、搞笑、给人轻松感。对方生气时也要带点可爱，不要真的让人不舒服。

---

## 十一、技术约束

- 全程中文
- 技术栈：Next.js（App Router）+ TypeScript + Tailwind CSS
- AI 能力：全部用扣子编程沙箱环境自带的（LLM 对话 + TTS 语音）
- 好感度由 LLM 在每轮回复中明确给出数值变化，前端负责计算和展示
- 尽可能增加动效

### 性能体验标准

- 选完场景后 1 秒内显示对方第一句话的文字
- 选项逐个飞入（stagger 动画）掩盖 LLM 生成等待
- 语音最晚在选项出现后 3 秒内可播放
- 后续每轮同理

---

## 十二、明确不做的事

- ❌ 不做自由文本输入（只用选择题）
- ❌ 不做用户注册登录
- ❌ 不做数据持久化（刷新页面重新开始）
- ❌ 不做 NSFW 内容
- ❌ 不做多人对战
- ❌ 不预设题库（全部动态生成）
- ❌ 不做用户对 AI 回复质量的反馈机制（无 👎 按钮、无纠错入口）
- ❌ 同一场景可重复游玩，每次由 LLM 重新生成，不缓存历史、不做难度递进

---

## 十三、成功标准

1. 选完场景后 1 秒内显示对方第一句话的文字，选项逐个飞入动画掩护生成等待，语音最晚在选项出现后 3 秒内可播放
2. 每轮 6 个选项，2 好 4 坏，坏选项里有让人笑出来的
3. 对方的回复有情绪层次，和前面对话连贯
4. 每轮对方的回复能播放语音，语音体现情绪变化
5. 好感度变化有动画反馈（加分绿色上升，减分红色下降）
6. 游戏结束可生成结果卡片图片 + 带场景参数的分享链接，用户想截图发朋友圈或分享给朋友
